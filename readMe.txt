TCS2 folder structure, why to use this?

1) App that we are developing is growing.
2) Folder Structure always helps in making the work easier
3) If we use the structure based on the feature then it will be easy to manage.
For ex: Let's say we have a functionality where we compare with the competitors so if the view,controllers are there in the same folder
then it will be helpful to developers to change only that part.
4) This folder structure will be helpful for the Team Lead so that he can give access to only the needed feature.This helps in
best organising and managing the project.
